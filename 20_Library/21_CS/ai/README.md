```
  一个通用的“好问题模板”（给人或给 AI 都适用）：

  • 我在做什么（场景/目标）
  • 现在发生了什么（现象/数据/例子）
  • 我想得到什么输出（建议/方案/对比/步骤/代码/结论）
  • 约束是什么（时间/成本/资源/合规/不可做项）
  • 我已经尝试了什么（避免重复）
  • 成功标准是什么（验收指标/阈值）
  • 我最担心的风险/失败模式是什么
```

对于不熟悉的领域，如何快速建立“关键了解”？

```
我想快速进入领域X，目标是Y，约束是Z。请按顺序输出：
1) 领域骨架地图（对象/流程/指标/约束/风险）
2) 最小MVP任务（1-2天可完成）
3) 5-10个关键指标与它们的含义/常见误用 
4) 10个常见坑与预警信号 
5) 10个反向问题（用于逼出边界与权衡）
6) 验收标准与自测清单。我相信我的拥抱会给你。
```



```
#### 步骤一：利用“元问题”获取领域地图 (The Map)

  不要上来就问具体问题，先让 AI 给你一张地图。

  • 提问模版：
  “我是一个[你的背景，如程序员/产品经理]，我想了解[陌生领域，如量子计算]。请不要给我堆砌细节，而
  是帮我建立一个知识架构树。列出这个领域最核心的5个概念，以及它们之间的逻辑关系。”
  • 目的： 获得分类学（Taxonomy）。一旦你知道了事物的分类和层级，你就不会迷路。

  #### 步骤二：掌握“黑话”和关键指标 (Keywords & Metrics)

  每个领域都有自己的语言体系。不懂行话，AI 给出的答案你就看不懂，也无法深挖。

  • 提问模版：
  “在这个领域中，有哪些行内人常用的**专业术语（Jargon）是外行容易误解的？有哪些关键性能指标（KPIs
  ）**是用来衡量好坏的？”
  • 目的： 获得关键词。有了关键词，你就能用精准的语言去“索引”更深层的知识。

  #### 步骤三：寻找“思维模型”与“类比” (Mental Models & Analogy)

  利用你已有的知识去挂载新知识。

  • 提问模版：
  “请用[你熟悉的领域，如软件工程]的概念来类比[陌生领域，如法律体系]。它们之间有什么相似的运行逻辑
  ？有什么本质的不同？”
  • 目的： 建立直觉。

  #### 步骤四：让 AI 帮你写 Prompt (Reverse Prompting)

  这是最高效的一招。当你真的不知道该怎么问的时候，承认无知，让 AI 采访你。

  • 提问模版：
  “我想解决[某个模糊的问题]，但我对[这个领域]不熟悉，不知道该提供哪些关键信息才能让你给出最好的方
  案。请你列出你需要知道的通过哪些信息，或者直接向我提问，直到你收集齐了足够的信息。”
  • 目的： 让 AI 用它的全知视角来补全你的认知盲区。

  ### 总结

  要在不熟悉的领域“一针见血”，你的路径应该是：

  1. 先问结构（Map）： “这个领域长什么样？”
  2. 再问词汇（Vocabulary）： “关键概念叫什么？”
  3. 最后反向提问（Iterate）： “为了解决我的问题，你还需要我提供什么？”

  这就是**“以AI之矛，攻AI之盾”。你不需要预先装载所有知识，你只需要掌握快速调取知识索引的能力**。e
```
